What They Build

  • Nexus Copilot: An open, vendor-agnostic design environment built on VS Code that uses LLMs tuned for IEC-61131-3 languages (Ladder Logic, Structured Text, etc.) to help engineers write, explain, and test PLC code¹
  • Coding Agents: State-of-the-art LLMs fine-tuned to natively output industrial control languages — not just general-purpose code¹
  • Ops Agents: AI assistants that continuously monitor PLCs, historian data, and CMMS history to analyze context, recommend next steps, and execute actions¹

  • Standards Customization: Organizations can define naming conventions, code standards, and upload reference documentation to customize the AI per-project¹

The Problem

"As a software engineer working on manufacturing systems at Tesla, I expected to have cutting-edge tools for manufacturing systems, just as we did for vehicle software. The reality was quite different! Instead, increasingly complex equipment is being built, operated, and maintained using aging, vendor-locked software tools and PLC platforms." — Chris Yuan, LinkedIn²
  • Aging toolchains: Industrial automation still relies on proprietary, decades-old IDE environments from Rockwell, Siemens, etc.
  • Vendor lock-in: Each PLC platform has its own tooling — no unified development environment
  • Labor shortage: Controls engineers are in critical short supply; experienced PLC programmers are retiring faster than they're being replaced
  • Rising complexity: Automated manufacturing equipment is getting more complex, but the tools haven't kept up
  • Cost of production: All of this drives up manufacturing costs and makes it harder to build products locally (reshoring challenge)
